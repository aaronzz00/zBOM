Core Design

The first editable version introduces an EBOM architecture domain store and a repository interface. The default repository is in-memory and seeded from mockEBOMArchitecture, but the store talks to the interface shape a backend adapter can later implement.

Edits happen as draft mutations against EBOM items: override field values, lock or unlock fields, add local child items, and reset/revert draft changes. Publishing creates a structured change package and advances local base status/revision semantics without touching useBOMStore.

Preserves

  • New EBOM domain separate from legacy BOMNode
  • Resolver remains pure and testable
  • Backend persistence can replace only the repository

Defers

  • Real authentication and approval routing
  • Server conflict resolution
  • Full release/change-control workflow
