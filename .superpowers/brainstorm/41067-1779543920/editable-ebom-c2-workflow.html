Workflow Rules

Editing starts from the resolved item list. Inherited rows can be overridden into local draft rows. Locked fields stay explicitly tracked in lockedFields. Local rows can be added under the selected parent. The right-side change package summarizes draft operations before publish.

Publish does not call a real backend in this phase. It sends the draft through the repository interface, records a local change package, clears dirty state, and updates local status/revision metadata so tests can assert the workflow contract.
